## The 6-Hour Daily "Summer Boot Camp" Schedule

[2 Hours: Deep Learning] ➔ [3 Hours: Active Building] ➔ [1 Hour: Portfolio & System Setup]

## Block 1: Deep Learning & Concept Mastery (2 Hours — Morning)

* What to do: Watch your tutorials or read documentation at maximum focus.
* The rule: Take notes, but do not just watch passively. Keep your code editor open and type along with every example.
* Topics to finish this month: DOM manipulation, Event Listeners, fetch() and Async/Await, and ES6+ features (array methods like .map() and .filter()).

## Block 2: Unguided Project Building (3 Hours — Afternoon)

* What to do: Close YouTube completely. Build small projects completely from scratch without following a video step-by-step.
* How to handle getting stuck: Use your AI assistant (like Cursor or ChatGPT) as a personal tutor. Do not ask it to write the code for you. Ask: "Why is my function returning undefined here?" or "Explain this error message to me."
* What to build this week: A fully functional digital calculator, a multi-step survey form with validation, or a countdown timer.

## Block 3: Portfolio, Git, & Ecosystem Setup (1 Hour — Evening)

* What to do: Treat this as your administrative hour to build your professional presence.
* Tasks to rotate through: Push your daily code to GitHub, polish the README files of your repositories, customize your LinkedIn profile, or read freelance job postings on Upwork to analyze what skills clients are actively paying for right now.

## How to Survive the High-Intensity Summer Pace

* Protect Your Laptop: With longer hours, ensure you are working in a cool environment and keep your devices backed up on GitHub hourly in case of sudden power cuts.
* The 50/10 Rule: Work for 50 minutes, then walk away from your screen for 10 minutes. High-intensity coding for 6 hours straight without breaks will lead to exhaustion by week two. [3, 4] 
* Build One Local Project Early: Use your extra time to find a family member or local business owner in your neighborhood who needs a simple informational website. Build it for free just to practice managing a real client.

## 📅 Your Day 1 to Day 7 Master Schedule

[2 Hours: Learn Concepts] ➔ [3 Hours: Build Without Videos] ➔ [1 Hour: Git & AI Debugging]

## 🗓️ Days 1 & 2: Mastering the DOM (Document Object Model)
Stop tracking data in the console. Learn how JavaScript talks directly to your HTML and CSS.

* Concept Focus (2 hrs): Target elements using document.querySelector(). Learn to modify content with .innerText and styles with .classList.add().
* Build Project (3 hrs): Create a Dark Mode Toggle and a Modal Pop-up window. Clicking a button must smoothly change the background color of the whole page.
* Admin/Git (1 hr): Set up a GitHub account if you haven't. Learn two commands: git commit and git push.

## 🗓️ Days 3 & 4: Event Listeners & User Input
Learn how to capture what a user types or clicks, and process that information dynamically.

* Concept Focus (2 hrs): Learn .addEventListener('click'), .addEventListener('input'), and how to prevent page reloads using e.preventDefault().
* Build Project (3 hrs): Build an ETB Currency Converter. Create an input field where a user types an amount in USD, and it instantly calculates and displays the value in Ethiopian Birr on the screen.
* Admin/Git (1 hr): Push your currency converter to GitHub. Write a short, clean README file explaining what the project does.

## 🗓️ Days 5 & 6: Working with Arrays & Dynamic Lists
This is the foundational logic you will use later when building modern React applications.

* Concept Focus (2 hrs): Master array methods: .map(), .filter(), and .forEach(). Learn how to create new HTML elements dynamically using document.createElement().
* Build Project (3 hrs): Build a To-Do List App. Users should be able to type a task, click "Add," see it join a list, and click a "Delete" button next to the task to remove it.
* Admin/Git (1 hr): Use your AI assistant (Cursor or VS Code AI) to review your To-Do list code. Ask it: "How can I make this JavaScript code cleaner and shorter?"

## 🗓️ Day 7: The Weekly Consolidation Review

* No New Concepts Today. Take your three mini-projects (Dark Mode, Currency Converter, To-Do List) and merge them into one single HTML file with a clean navigation bar.
* Practice fixing your own bugs. If something breaks, read the red error text in your browser console, paste it into your AI assistant, and learn exactly what caused the break.

------------------------------
## 💡 3 Golden Rules for This Week

   1. Never Copy-Paste: Even if you are looking at a code example, type it out character by character. This builds crucial muscle memory.
   2. Handle the Outages: Download your learning videos or documentation pages to your phone or laptop ahead of time so you can keep writing code even if your internet drops out.
   3. No Tutorial Pumping: If a video is 30 minutes long, it should take you at least 90 minutes to finish it because you are constantly pausing to test the code yourself.
